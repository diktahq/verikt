## See It in Action

```bash
# Interactive wizard — guides you through every choice
archway new my-service

# Or go non-interactive
archway new my-service \
  --arch hexagonal \
  --cap platform,bootstrap,http-api,mysql,docker \
  --module github.com/myorg/my-service \
  --no-wizard
```

You get a production-ready service with:
- Hexagonal architecture with strict dependency rules
- REST API with Chi router and RFC 7807 errors
- MySQL with connection pooling and health checks
- Structured logging, OpenTelemetry, PII redaction
- Bootstrap pattern for testable dependency wiring
- Docker Compose for local development
- `archway.yaml` for architecture enforcement

<Aside type="tip">
The smart wizard suggests what you're missing. Adding HTTP? You probably want rate limiting, CORS, and auth too. 18 suggestion rules help you build complete services.
</Aside>
