## Using `archway add`

The simplest way to add capabilities to an existing project:

```bash
archway add redis
```

This will:
1. Validate the capability exists and check for conflicts
2. Auto-resolve transitive dependencies
3. Render new files (skipping any that already exist)
4. Update your `archway.yaml`
5. Regenerate AI agent context via `archway guide`

### Adding Multiple Capabilities

```bash
archway add kafka-consumer observability redis
```

### Transitive Dependencies

Some capabilities require others. archway resolves these automatically:

```bash
archway add bff
# Output:
# Adding capabilities:
#   + http-api (auto-dependency)
#   + bff
```

### Conflict Detection

If a capability conflicts with one already installed, archway will stop and tell you:

```bash
archway add grpc
# Error: capability "grpc" conflicts with "http-api"
```

<Aside type="tip">
Use `archway check` after adding capabilities to verify your project still passes all architecture rules.
</Aside>

## At Scaffold Time

You can also include capabilities when creating a new project:

```bash
archway new my-service \
  --arch hexagonal \
  --cap platform,bootstrap,http-api,mysql,redis,docker
```

Or use the interactive wizard to explore all 63 capabilities with descriptions and smart suggestions:

```bash
archway new my-service
```

## Understanding Capability Structure

Each capability provides:

1. **Template files** — Source files rendered into your project
2. **Partials** — Code snippets that wire into `bootstrap.go`
3. **Config fields** — Struct fields added to `config/config.go`
4. **Manifest** — Metadata about requirements, suggestions, and conflicts

### Example: What `mysql` Provides

```
capabilities/mysql/
├── capability.yaml              # name, requires, suggests, conflicts
├── files/
│   └── adapter/mysqlrepo/
│       └── connection.go.tmpl   # Connection pooling setup
└── _partials/
    ├── main_imports.go.tmpl     # import "github.com/org/svc/adapter/mysqlrepo"
    ├── main_init.go.tmpl        # db, err := mysqlrepo.NewConnection(...)
    └── main_shutdown.go.tmpl    # app.OnShutdown("mysql", ...)
```

## Checking Compatibility

Before adding a capability, check its `capability.yaml` for:

- **`requires`** — Other capabilities that must be present (auto-resolved by `archway add`)
- **`suggests`** — Recommended companions
- **`conflicts`** — Capabilities that can't coexist

```yaml
# Example: auth-jwt requires http-api
name: auth-jwt
requires:
  - http-api
suggests: []
conflicts: []
```

## Manual Addition

If you prefer to add capabilities by hand:

<Steps>

1. **Add the source files**

   Look at the capability's `files/` directory for what to create. For example, adding Redis means creating `adapter/redisrepo/connection.go`.

2. **Update config**

   Add the capability's config struct and fields to `config/config.go`:

   ```go
   type RedisConfig struct {
       Addr     string `yaml:"addr"`
       Password string `yaml:"password"`
       DB       int    `yaml:"db"`
   }
   ```

3. **Wire into bootstrap**

   Add initialization and shutdown code to `internal/bootstrap/bootstrap.go`.

4. **Update archway.yaml**

   Add the capability to your project's capability list:

   ```yaml
   capabilities:
     - platform
     - bootstrap
     - http-api
     - mysql
     - redis  # Added
   ```

5. **Regenerate guide**

   ```bash
   archway guide --target claude
   ```

</Steps>

<Aside type="note">
`archway add` does all of these steps automatically. Manual addition is useful when you need custom wiring or want to understand the internals.
</Aside>
