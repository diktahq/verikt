You've started enough Go services to know what the first two hours look like: create the module, set up config loading, wire the logger, decide where middleware goes, argue with yourself about where to put error types, copy-paste the graceful shutdown code from the last project. By the time you write your first handler, you've already made 20 structural decisions — and none of them are specific to the problem you're actually solving.

archway handles that part.

## What archway does for Go developers

- Scaffolds idiomatic Go project structure for hexagonal, layered, clean, or flat architectures — with the dependency rules already encoded
- Composes 63 modular capabilities (HTTP API, PostgreSQL, JWT auth, circuit breaker, OpenTelemetry, etc.) into a working project — no dead imports, no unused code
- Generates production patterns correctly: RFC 7807 errors, `slog` structured logging with PII redaction, OpenTelemetry instrumentation, Chi middleware chain, graceful shutdown via `lifecycle.App`
- Puts an `archway.yaml` in the project so `archway check` can catch layer violations before they compound

## The problem it solves

Every Go developer has a "starter" project they copy from. The problem is that starter projects age. Three months later it has the tech debt of whoever wrote it, the library versions from whenever it was last touched, and half the patterns are inconsistent because it was built incrementally rather than designed. You copy it anyway because starting from scratch is worse.

The deeper issue is that project structure is load-bearing. Where you put things matters — it shapes how the codebase grows, how easy it is to test, and how confidently a new engineer can navigate it. archway doesn't just scaffold files; it encodes the architectural decisions (hexagonal layers and their dependency rules, bootstrap pattern for testable dependency wiring, capability-aware config structs) so the structure is correct from the first commit.

## How it works

```bash
# The command a Go developer reaches for most
archway new my-service --arch hexagonal --cap platform,bootstrap,http-api,health,observability,postgres,migrations,docker
```

You get a compilable Go module with Chi router, pgx PostgreSQL connection, golang-migrate runner, OpenTelemetry setup, structured logging, and Docker Compose — all wired together with the correct layer boundaries. The wizard variant (`archway new my-service`) walks you through each choice interactively and flags capability conflicts before you commit.
