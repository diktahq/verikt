---
title: Who Uses archway?
description: archway for Go developers, microservices teams, backend engineers, platform teams, and AI coding agents — see how each gets value.
---

archway is used by different people for different reasons. A Go developer cares about idiomatic project structure and not wiring boilerplate from scratch. A platform team cares about enforcing that 12 services actually follow the architecture they agreed on. An AI coding agent needs architectural context so it doesn't invent its own conventions on every new file.

The tool is the same. The problem it solves for each is different.

| Persona | Main benefit | Page |
|---------|-------------|------|
| **Go Developers** | Production-ready Go project structure in one command — hexagonal, layered, or clean, with the capabilities you actually need | [archway for Go Developers](/for/go-developers/) |
| **Microservices Teams** | Consistent structure across every service in your org — same patterns, same capabilities, same enforcement rules | [archway for Microservices Teams](/for/microservices-teams/) |
| **Backend Engineers** | Skip the setup. Get structured logging, OpenTelemetry, graceful shutdown, and RFC 7807 errors wired correctly from day one | [archway for Backend Engineers](/for/backend-engineers/) |
| **Platform & SRE Teams** | Enforce architecture rules in CI with `archway check` — 11 AST-based detectors catch drift before it ships | [archway for Platform Teams](/for/platform-teams/) |
| **AI Coding Agents** | Feed Claude, Cursor, Copilot, or Windsurf your full architecture context with `archway guide` — agents write correct code from the first line | [archway for AI Coding Agents](/for/ai-coding-agents/) |
