`docker` adds a `docker-compose.yml` to your Go service for running local development dependencies with a single command.

## What You Get

- `docker-compose.yml` with service dependencies
- `.env.example` for environment variables

## When to Use

Use `docker` whenever your service depends on external infrastructure — databases, caches, message brokers, or third-party services. Docker Compose lets every developer spin up the full stack locally without manual installation.

## Add to Your Service

```bash
archway new my-service --cap docker
# or add to an existing service:
archway add docker
```
