`postgres` wires PostgreSQL into your Go service using the pgx driver, with connection pooling and a repository scaffold ready to use.

## What You Get

- Connection setup with pgx driver
- Pool configuration
- Health checks
- Repository scaffold

## When to Use

Use `postgres` for relational data with SQL querying, transactional workloads, or when you need PostgreSQL-specific features like JSONB, full-text search, or advanced indexing. Pair with `migrations` for schema management and `uuid` for database-friendly primary keys.

## Requirements & Suggestions

**Suggests:** [`migrations`](/capabilities/migrations/), [`docker`](/capabilities/docker/), [`uuid`](/capabilities/uuid/)

## Add to Your Service

```bash
archway new my-service --cap postgres
# or add to an existing service:
archway add postgres
```
