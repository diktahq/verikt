`redis` wires Redis into your Go service with a configured connection and repository pattern scaffold.

## What You Get

- Connection management
- Repository pattern scaffold

## When to Use

Use `redis` for caching, session storage, rate limiting backends, or pub/sub. It pairs well with `idempotency` (key storage) and `rate-limiting` (token bucket backend).

## Requirements & Suggestions

**Suggests:** [`docker`](/capabilities/docker/)

## Add to Your Service

```bash
archway new my-service --cap redis
# or add to an existing service:
archway add redis
```
